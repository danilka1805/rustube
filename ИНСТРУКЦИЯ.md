### (Написано для России, с нуля, без технического опыта)

---

## ⚠️ Сначала про Cloudflare в РФ

Cloudflare **частично заблокирован** в России — Роскомнадзор периодически блокирует
отдельные диапазоны их IP. Это означает, что у части пользователей сайт может не открыться.

**Решение:** Вместо Cloudflare используем **дешёвый российский VPS** (~100–150 ₽/мес)
как «посредник». Он будет принимать запросы от пользователей и пробрасывать их на ваш NUC.

Два пути:
- **Путь A — Cloudflare (бесплатно, может не работать у части РФ-пользователей)**
- **Путь Б — Российский VPS (100–150 ₽/мес, работает у всех в РФ)** ← рекомендую

# ЧАСТЬ 1 — АККАУНТЫ (создаём один раз)

## 1.1 Supabase — база данных и превью

Supabase работает из России нормально (серверы в США, но не заблокированы).

1. Откройте **https://supabase.com**
2. Нажмите **"Start your project"**
3. Войдите через **GitHub** (или создайте аккаунт GitHub отдельно на github.com)
4. Нажмите **"New project"**
5. Заполните:
   - **Organization**: оставьте как есть
   - **Name**: `p2pvideo` (любое слово без пробелов)
   - **Database Password**: придумайте надёжный пароль, **запишите его**
   - **Region**: выберите `Frankfurt (EU Central)` — ближайший к РФ
6. Нажмите **"Create new project"** — ждите ~2 минуты

### Создаём таблицы в базе данных:
1. В левом меню нажмите **"SQL Editor"**
2. Нажмите **"New query"**
3. Вставьте содержимое файла `supabase_schema.sql` целиком
4. Нажмите **"Run"** (или Ctrl+Enter)
5. Должно появиться: `Success. No rows returned`

### Создаём хранилище для превью:
1. В левом меню нажмите **"Storage"**
2. Нажмите **"New bucket"**
3. Name: `thumbnails`
4. Включите тумблер **"Public bucket"** ✅
5. Нажмите **"Save"**

### Копируем ключи (понадобятся позже):
1. В левом меню нажмите **"Settings"** (шестерёнка внизу)
2. Нажмите **"API"**
3. Скопируйте и сохраните в блокнот:
   - **Project URL** — выглядит как `https://abcdefgh.supabase.co`
   - **anon public** (под надписью "Project API keys") — длинная строка начинается с `eyJ`

# ЧАСТЬ 2 — СЕРВЕР НА NUC

## 2.1 Установка нужных программ

Откройте терминал на NUC и выполните по одной строке:

```bash
# Обновляем список пакетов
sudo apt update

# Устанавливаем Node.js 20 (официальный способ)
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# Устанавливаем FFmpeg (для сжатия видео)
sudo apt install -y ffmpeg

# Проверяем что всё встало:
node --version   # должно показать v20.x.x
ffmpeg -version  # должно показать версию ffmpeg
```

## 2.2 Копируем файлы сервера на NUC

Скопируйте папку `server/` на NUC. Например, через флешку или:
```bash
# Создаём папку
mkdir -p ~/p2pvideo-server
cd ~/p2pvideo-server
```
Положите туда файлы `server/index.js` и `server/package.json`.

## 2.3 Устанавливаем зависимости и запускаем

```bash
cd ~/p2pvideo-server

# Устанавливаем зависимости Node.js
npm install

# Тестовый запуск (убедиться что работает)
node index.js
```

Должно появиться:
```
🚀 P2PVideo сервер запущен на порту 3000
📁 Папка видео: /home/.../p2pvideo-server/videos
⏳ Видео хранится 3 дня, затем удаляется автоматически
```

Нажмите **Ctrl+C** чтобы остановить — настоящий запуск сделаем после туннеля.

# ЧАСТЬ 3 — ТУННЕЛЬ (выберите один путь)

## 🇷🇺 ПУТЬ Б — Российский VPS (рекомендую)

### 3.1 Берём VPS

**Рекомендую Timeweb Cloud** — российская компания, дёшево, быстро:

1. Откройте **https://timeweb.cloud**
2. Зарегистрируйтесь (нужен телефон РФ или email)
3. Нажмите **"Создать сервер"**
4. Выберите конфигурацию:
   - **Операционная система**: Ubuntu 22.04
   - **Тариф**: самый дешёвый (1 CPU, 1 ГБ RAM) — ~90–120 ₽/мес
   - **Регион**: Россия
5. Нажмите **"Заказать"**
6. Через 1–2 минуты в письме придут:
   - IP-адрес сервера (например `185.x.x.x`)
   - root-пароль

**Альтернативы VPS:** REG.RU, Beget, Selectel, VK Cloud (у них тоже есть дешёвые серверы)

### 3.2 Настраиваем VPS как прокси

Подключитесь к VPS по SSH. В Windows используйте **PuTTY** (putty.org) или Windows Terminal:
```
ssh root@185.X.X.X
# вводите пароль из письма
```

На VPS выполните:
```bash
# Обновляем систему
apt update && apt upgrade -y

# Устанавливаем Nginx
apt install -y nginx

# Создаём конфиг прокси
nano /etc/nginx/sites-available/p2pvideo
```

В редакторе nano вставьте (замените `185.X.X.X` на IP вашего VPS):
```nginx
server {
    listen 80;
    server_name 185.X.X.X;

    # Большие файлы (видео до 10 ГБ)
    client_max_body_size 10240m;
    client_body_timeout 600s;
    proxy_read_timeout 600s;
    proxy_send_timeout 600s;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_buffering off;

        # CORS для браузера
        add_header Access-Control-Allow-Origin *;
        add_header Access-Control-Allow-Methods 'GET, POST, OPTIONS';
        add_header Access-Control-Allow-Headers '*';
        if ($request_method = OPTIONS) { return 204; }
    }
}
```

Нажмите **Ctrl+O**, Enter (сохранить), **Ctrl+X** (выйти).

```bash
# Активируем конфиг
ln -s /etc/nginx/sites-available/p2pvideo /etc/nginx/sites-enabled/
nginx -t          # проверка — должно быть "syntax is ok"
systemctl restart nginx

# Открываем порты
ufw allow 22
ufw allow 80
ufw allow 443
ufw enable
```

### 3.3 Пробрасываем NUC через SSH-туннель

**На NUC** (не на VPS!) выполните. Замените `185.X.X.X` на IP вашего VPS:

```bash
# Установка autossh (для автоматического переподключения)
sudo apt install -y autossh

# Тест (запустите в терминале, убедитесь что работает):
ssh -R 3000:localhost:3000 root@185.X.X.X -N
# Если всё ок — подключится без ошибок. Ctrl+C для выхода.
```

Теперь сделаем автозапуск — чтоб туннель поднимался при старте NUC:

```bash
sudo nano /etc/systemd/system/p2pvideo-tunnel.service
```

Вставьте (замените `185.X.X.X` и `ВАШ_ПОЛЬЗОВАТЕЛЬ`):
```ini
[Unit]
Description=P2PVideo SSH Tunnel
After=network.target

[Service]
User=ВАШ_ПОЛЬЗОВАТЕЛЬ
ExecStart=/usr/bin/autossh -M 20000 -N -o "StrictHostKeyChecking=no" \
  -o "ServerAliveInterval=30" -o "ServerAliveCountMax=3" \
  -R 3000:localhost:3000 root@185.X.X.X
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
# Автозапуск туннеля
sudo systemctl enable --now p2pvideo-tunnel
sudo systemctl status p2pvideo-tunnel  # должно быть "active (running)"
```

Также настроим сервер Node.js на автозапуск:
```bash
sudo nano /etc/systemd/system/p2pvideo.service
```
```ini
[Unit]
Description=P2PVideo Node Server
After=network.target

[Service]
User=ВАШ_ПОЛЬЗОВАТЕЛЬ
WorkingDirectory=/home/ВАШ_ПОЛЬЗОВАТЕЛЬ/p2pvideo-server
ExecStart=/usr/bin/node index.js
Restart=always
RestartSec=5
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
```
```bash
sudo systemctl enable --now p2pvideo
sudo systemctl status p2pvideo
```

**Ваш VIDEO_SERVER_URL = `http://185.X.X.X`**

> **Опционально:** Можно прикрепить домен. Берёте бесплатный домен на **freenom.com** или
> покупаете `.ru` за 150 ₽/год на REG.RU, направляете на IP VPS, и используете как URL.

## ☁️ ПУТЬ А — Cloudflare Tunnel (бесплатно, если работает у вас)

Проверьте сначала, доступен ли Cloudflare с вашего провайдера:
```bash
curl -I https://cloudflare.com
# Если видите "HTTP/2 200" — работает. Если timeout — не работает.
```

Если работает:
```bash
# Установка cloudflared
wget https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb
sudo dpkg -i cloudflared-linux-amd64.deb

# Запуск туннеля (одноразовый URL, бесплатно, без аккаунта)
cloudflared tunnel --url http://localhost:3000
```

Вы увидите URL вида `https://random-name.trycloudflare.com` — это ваш `VIDEO_SERVER_URL`.

**Минус:** URL меняется при каждом перезапуске. Для постоянного URL нужен аккаунт Cloudflare
и Named Tunnel — это бесплатно, но сложнее в настройке.

## 5.1 Netlify (самый простой способ)

1. Откройте **https://netlify.com** → войдите

# ЧАСТЬ 6 — ФИНАЛЬНАЯ ПРОВЕРКА

## Чеклист

| Шаг | Проверка |
|-----|----------|
| Supabase | Откройте `https://ВАШЕ.supabase.co` — должна открываться панель |
| Таблицы | В Supabase → Table Editor должны быть таблицы `videos`, `likes`, `comments` |
| NUC сервер | `curl http://localhost:3000/stats` → должно вернуть `{"videos":0,"totalGB":"0.00"}` |
| Туннель | `curl http://185.X.X.X/stats` → то же самое, но с VPS |
| Сайт | Открыть ваш netlify/github-pages URL — должна открыться главная |
| Загрузка | Попробовать загрузить видео — должен появиться прогресс |
| Просмотр | Нажать на видео — должно начать воспроизводиться |

## Частые проблемы

**Сайт не может подключиться к серверу (CORS ошибка)**
→ Добавьте ваш netlify URL в массив `ALLOWED_ORIGINS` в `server/index.js`:
```javascript
'https://your-site.netlify.app',
```
Перезапустите: `sudo systemctl restart p2pvideo`

**Видео долго грузится**
→ Нормально. FFmpeg транскодирует на NUC. Прогресс можно смотреть:
`sudo journalctl -u p2pvideo -f`

**SSH туннель отваливается**
→ Проверьте: `sudo systemctl status p2pvideo-tunnel`
→ Перезапустить: `sudo systemctl restart p2pvideo-tunnel`

**Supabase не подключается**
→ Проверьте что скопировали `anon` ключ, а не `service_role`
